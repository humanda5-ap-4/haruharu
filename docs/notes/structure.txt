[사용자 입력]
   ↓
[NLP + 시맨틱 엔진]
   ↓
(JSON 출력: intent + entities)
   ↓
[기능 모듈 판단 및 실행]
   ↓
[응답 시나리오 or 데이터 조회 → 사용자 응답]


intent	연결될 기능 모듈
운동추천	건강/운동 (팀원 1)
식단추천	식단/영양 (팀원 2)
감정상태확인	마음건강 (팀원 5)
소비기록조회	생활/소비 (팀원 4)

각 기능 모듈에서는 입력된 entities 값을 기반으로 맞춤형 응답을 생성해야 합니다.

예: 운동추천 기능 모듈에서

json
복사
편집
{
  "intent": "운동추천",
  "entities": {
    "운동종류": "달리기",
    "목표": "근육증가"
  }
}
→ 이 정보로 ‘달리기 + 근육 증가’에 적합한 운동 루틴을 DB에서 조회하거나 API 호출

3. 모듈 처리 결과 반환 예시
기능 모듈은 최종 응답 템플릿 또는 추천 결과를 반환해야 합니다.

json
복사
편집
{
  "response_type": "text",
  "text": "근육 증가를 위한 달리기 운동 루틴을 추천해드릴게요: 인터벌 러닝 20분 + 보강운동"
}
또는 UI 연동용 구조:

json
복사
편집
{
  "response_type": "card",
  "title": "오늘의 추천 운동",
  "items": [
    {"name": "인터벌 달리기", "duration": "20분"},
    {"name": "스쿼트", "duration": "3세트"}
  ]
}
